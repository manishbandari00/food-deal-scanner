# 🍜 Food Deal Scanner

A terminal-based food deal scanner that simulates a live food delivery system for Kompally, Hyderabad. It finds the **cheapest restaurant combo within your budget**, auto-applies the best coupon, and shows a full bill breakdown — all from your terminal.

Built as a real-world Python learning project. No frameworks, no cloud — just Python, a local server, and a live data engine running in 3 terminals.

## 🧩 How It Works

The system has 3 components running simultaneously:

```
Terminal 1: python data_engine.py    ← generates live_data.json every 30 mins
Terminal 2: python food_server.py    ← serves browser UI at localhost:8765
Terminal 3: python deal_scanner.py   ← main scanner (set budget → scan → get deal)
Browser:    http://localhost:8765    ← visual restaurant browser
```

All 3 tools read the **same `live_data.json`** — so the terminal and browser always show identical live data.

## ✨ Features

### Terminal Scanner
- **17 food categories** — Biryani, Pizza, Burger, Momos, Dosa, Kebabs, and more
- **Smart veg/non-veg filter** — auto-detects what's available per cuisine; asks only when needed
- **Mix mode** — order 1 Veg + 1 Non-Veg of the same cuisine (e.g. 2 Pizzas split)
- **Full billing engine** — item total + delivery + GST (5%) + platform fee − best coupon
- **Best Match panel** — shows cheapest open restaurant with complete itemized bill
- **All Items table** — every matching item across all restaurants, sorted open-first then cheapest, then by rating
- **Split Order logic** — if no single restaurant has all your cuisines, suggests the best split
- **Background monitor** — auto-scans every X minutes, sends desktop notification when deal found
- **Smart budget validation** — budget defaults to "not set"; warns you before scanning if not configured

### Data Engine
- Randomizes prices ±15% every 30 minutes (simulates real-world price changes)
- Randomly expires and injects new offers (pool of 12 offer types)
- Flips restaurant open/closed status (20% chance per cycle)
- Drifts ratings slightly (±0.2, clamped 3.0–5.0)

### Browser UI
- Visual restaurant browser served at `localhost:8765`
- Shows menus, offers, open/closed status
- Veg filter toggle, dish search, cart, coupon application, bill calculator
- Auto-refreshes every 30 mins to match data engine

## 🚀 Setup & Run

### 1. Clone the repo
```bash
git clone https://github.com/YOUR_USERNAME/food-deal-scanner.git
cd food-deal-scanner
```

### 2. Install dependencies
```bash
pip install rich schedule plyer
```

### 3. Run all 3 components (3 separate terminals)

**Terminal 1 — Data Engine** (run this first)
```bash
python data_engine.py
```
This generates `live_data.json`. Keep it running.

**Terminal 2 — Food Server**
```bash
python food_server.py
```
Then open Chrome → `http://localhost:8765` for the visual browser.

**Terminal 3 — Deal Scanner**
```bash
python deal_scanner.py
```
Then:
1. Press `4` → set your budget
2. Press `3` → pick cuisines + quantities + veg preference
3. Press `1` → scan and get your best deal

## 💡 Key Design Decisions

- **Static curated categories** instead of raw restaurant tags — restaurant tags are vague and overlapping
- **Pasta matches item name only** — prevents the "Pasta & Sides" category from pulling in Garlic Bread
- **Kebabs excludes pizza items** — "Paneer Tikka Pizza" won't appear in Kebab results
- **Budget defaults to None** — forces you to consciously set it before scanning
- **Sort order: OPEN first, then cheapest, then highest rated** — a closed restaurant never wins Best Match over an open one
